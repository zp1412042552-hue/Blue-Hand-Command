# AI 队友行为系统设计

## 一、设计哲学

- **两个 AI 队友首先是保护者，然后是执行者。**
- 不指挥时：在玩家 5 米范围内自由巡逻警戒。
- 指挥时：通过手势选择并下达命令，执行完毕后回到自由巡逻。
- 发现敌人：自动攻击，不需要玩家指挥。

## 二、手势识别原则（与 TacticalGestureCommandSystem 保持一致）

AI 行为系统的手势输入由 `CommandGestureComponent` 驱动。手势系统遵循以下原则：

- **形态物理隔离**：无空间限制，仅靠手势独特性 + 0.5 秒时间过滤防误触
- **上下文相关**：同一个 👇 手势根据 Raycast Tag 自动切换命令
- **状态机被动激活**：命令手势只在选中队友后才激活检测

> 完整手势系统设计见 [TacticalGestureCommandSystem.md](TacticalGestureCommandSystem.md)

## 三、3 层行为状态

```
┌──────────────────────────────────────────────┐
│ 第1层：自由巡逻（默认状态）                    │
│  队友在玩家 5 米范围内游走/警戒               │
│  面朝外部，保护玩家                           │
│  发现敌人 → 自动攻击                         │
├──────────────────────────────────────────────┤
│ 第2层：命令等待（玩家用手势选中了队友）          │
│  ☝️ 伸1指 ≥ 0.5秒 → 队友A 高亮选中            │
│  ✌️ 伸2指 ≥ 0.5秒 → 队友B 高亮选中            │
│  ①② 连做 → 全选小队                          │
│  选中后队友停止移动，面向玩家等待命令          │
│  超时 2.0 秒无动作 → 自动清空选择              │
├──────────────────────────────────────────────┤
│ 第3层：执行命令（玩家下达了具体指令）          │
│  执行完毕后 → 回到第1层                       │
└──────────────────────────────────────────────┘
```

## 四、5 大手势与 AI 行为映射

### 4.1 选人手势（全局静默检测，任意位置均可触发）

| # | 手势 | 功能 | 触发条件 | AI 反应 |
|---|------|------|---------|---------|
| ① | ☝️ 单指指天 | **锁定队友 A** | 保持 ≥ 0.5 秒 | 队友 A 蓝色高亮，耳机"哔"，HUD 亮起 A |
| ② | ✌️ V 字指天 | **锁定队友 B** | 保持 ≥ 0.5 秒 | 队友 B 黄色高亮，耳机"哔"，HUD 亮起 B |

💡 **多选/全选逻辑**：锁定 A 后手不放下，紧接着做 ② → HUD 同时亮起 A 与 B → 判定为"全选小队"，后续命令对两人同时生效。

### 4.2 命令手势（选中队友后才激活检测）

| # | 手势 | 功能 | Raycast 上下文判定 | AI 行为 |
|---|------|------|-------------------|---------|
| ③ | 👇 万能行动命令 | 指向目标执行战术行动 | `Floor` → 战术移动到该坐标并寻找掩体 | 队友走至目标点 → 自动寻找最近掩体 → 进入警戒 |
| | | | `Door` → 突击破门/C4 | 队友前往门前 → 踢门/C4 → 进入房间搜索 → 反馈"安全"或"发现目标" |
| | | | `Enemy` → 全队集火 | 队友将该敌人列为首要目标 → 集中火力消灭 |
| ④ | ✋ 万能战术状态 | 原地战术姿态切换 | 无需射线 | 队友就地寻找最近掩体 → 高阶戒备 → 允许自主索敌并开火 |

### 4.3 全局召回手势（无需选人，任何时候生效）

| # | 手势 | 功能 | AI 行为 |
|---|------|------|---------|
| ⑤ | 👊 握拳拉回胸口 | **全队集合保卫** | 所有队友立刻放弃当前一切任务 → 以战术 V 字阵型向玩家靠拢 → 贴身护卫 |

## 五、底层交互状态机

```
[状态 1：日常战斗]
       │
       ▼ (识别到 ☝️ 或 ✌️ 并保持 0.5 秒)
[状态 2：选择目标激活] ──► 耳机"哔" ──► 开启 👇 和 ✋ 的检测开关
       │
       ├─► (做出 👇 或 ✋) ───────► [执行命令] ──► 自动退回 [状态 1]
       ├─► (检测到 动态👊) ──────► [全队集结] ──► 自动退回 [状态 1]
       └─► (超时超过 2.0 秒无动作) ─► [选择清空] ──► 自动退回 [状态 1]
```

- **状态 1**：仅开启 ☝️、✌️、👊 的特征捕捉。玩家正常射击、换弹，手部偶然做出 👇 或 ✋ 系统完全不理
- **状态 2**：关闭选人检测，开启射线 + 命令手势检测。HUD 高亮选中队友
- **状态 3**：命令触发瞬间关闭所有检测，数据清空，强制退回状态 1。超时 2.0 秒自动清空

## 六、Behavior Tree 逻辑（选择器优先级）

```
Root (Selector)
│
├── [1] 执行命令（最高优先级）
│   条件：CommandType != None
│   动作：执行对应命令
│   完成：CommandType = None，回到巡逻
│
├── [2] 战斗（次优先级）
│   条件：EnemyActor != None
│   动作：面向敌人 → 开火
│
├── [3] 自由巡逻（默认）
│   ├── 距离玩家 > 5米 → 走回玩家身边
│   └── 距离玩家 ≤ 5米 → 随机游走 / 站立警戒
```

## 七、数据通信（Blackboard 条目）

```
Blackboard 条目（每个队友独立）：
├── SelfActor               本身引用
├── PlayerActor             玩家引用
├── CommandType             (None / MoveTo / Breach / FocusFire / WatchAndEngage / Recall)
├── TargetLocation          目标位置（指地面 / 门前 / 敌人位置）
├── TargetDoor              目标门（破门时）
├── TargetEnemy             目标敌人（集火时）
├── EnemyActor              当前敌人
├── bSelected               是否被玩家选中
└── bAllSelected            是否全选模式
```

手势组件只负责：**识别姿势 → 修改 Blackboard → Behavior Tree 自动响应**

## 八、关键技术决策

| 决策 | 方案 | 理由 |
|------|------|------|
| 手势识别 | 特征点角度匹配 + 时间滤波 | 通用 OpenXR 手部追踪，无需 Oculus 专有插件 |
| 环境检测 | Line Trace（射线检测）by Channel/Tag | UE 内置，性能开销极低 |
| 寻路 | NavMesh（导航网格） | UE 内置，自动绕障，适用室内环境 |
| AI 决策 | Behavior Tree | UE 可视化节点搭建，无需手写复杂逻辑 |
| 数据通信 | Blackboard | UE 内置，与 Behavior Tree 天然配合 |
| 特殊动作 | Montage 动画（踢门/C4） | 播放动画序列，不需要 IK 或物理 |
| 代码量 | 只需要写 CommandGestureComponent | AI 部分全用 UE 编辑器搭节点 |

## 九、房间搜索路径点（SearchPoints）

```
每个房间预先放置搜索点：
┌──────────────────────┐
│                      │
│  ①                   │  ① → 左侧角落
│           ②          │  ② → 中央
│                      │  ③ → 右侧角落
│                ③     │  ④ → 目标物附近
│                      │
│        ④            │
│                      │
└──────────┬───────────┘
         门
         
队友进入房间后按顺序遍历搜索点。
到达每个点 → 短暂停留 → 扫描周围 → 移动到下一个。
全部遍历完毕 → 反馈"区域安全"或"发现目标"。
```

## 十、队友保护玩家（自由巡逻策略）

```
① 采样玩家位置
② 计算离玩家距离
③ 如果 > 5 米 → 走回玩家身边
④ 如果 ≤ 5 米 → 选择一个新的巡逻点（在 5 米半径内）
⑤ 走到巡逻点 → 站定 → 朝向外部警戒 → 停留数秒
⑥ 重复
```

警戒时面朝外部（面朝远离玩家的方向），确保玩家背后和侧面有人看着。
