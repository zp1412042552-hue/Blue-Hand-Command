# 任务评价体系

## 一、评价维度

每个维度 0~100 分，加权计算总分。

### 维度1：任务完成度（权重 30%）

```
完成主要目标（清理区域/找到目标）    → +50分
完成所有次要目标                     → +20分
清理所有房间                         → +15分
无人发现（潜行加成）                → +15分

如果主要目标失败 → 直接 D 级，不计算总分
```

### 维度2：队友存活与状态（权重 25%）

```
两人都存活且健康（血量>80%）     → 100分
两人都存活但受伤（血量<50%）     → 70分
一人阵亡                         → 40分
两人阵亡                         → 0分

额外加减：
  给队友下达有效指令 ≥ 5次       → +10分
  从未指挥队友                   → -20分
```

### 维度3：战斗效率（权重 20%）

```
击杀率 = 击杀数 / 总敌人数
  100% 击杀                      → 100分
  80%~99%                        → 80分
  50%~79%                        → 60分
  <50%                           → 30分

命中率 = 命中数 / 开火数
  >60%                           → 100分
  40%~60%                        → 70分
  20%~39%                        → 40分
  <20%                           → 10分

（取击杀率+命中率的平均值）
```

### 维度4：自身表现（权重 15%）

```
玩家受伤次数
  0次（无伤）                     → 100分
  1~2次                          → 75分
  3~5次                          → 45分
  >5次                           → 15分

队友伤害（误伤队友）              
  0次                            → 保持
  有误伤                          → 直接扣50分
```

### 维度5：资源管理（权重 10%）

```
弹药利用率 = (击杀数 / 总消耗子弹数)
  每杀一个敌人用 <5 发            → 100分
  每杀一个敌人用 5~10 发          → 70分
  每杀一个敌人用 10~20 发         → 40分
  每杀一个敌人用 >20 发           → 10分

时间效率：
  快于目标时间                    → +10分
  慢于目标时间2倍                 → -10分
```

## 二、总分与等级

```
总分 = 任务完成×30% + 队友状态×25% + 战斗效率×20% + 自保×15% + 资源×10%
```

| 等级 | 分数 | 称号 | 颜色 |
|------|------|------|------|
| **S** | 90~100 | 精英指挥官 | ★ 金色 |
| **A** | 75~89 | 优秀指挥官 | ★ 银色 |
| **B** | 60~74 | 合格指挥官 | ★ 蓝色 |
| **C** | 40~59 | 需要改进 | ★ 白色 |
| **D** | 0~39 | 严重失误 | ☆ 灰色 |

> 特殊规则：主要目标失败 → 直接 D，不计算总分

## 三、C++ 数据结构

```cpp
UENUM(BlueprintType)
enum class EMissionRank : uint8
{
    D UMETA(DisplayName = "D"),
    C UMETA(DisplayName = "C"),
    B UMETA(DisplayName = "B"),
    A UMETA(DisplayName = "A"),
    S UMETA(DisplayName = "S")
};

USTRUCT(BlueprintType)
struct FMissionScoreBreakdown
{
    GENERATED_BODY()

    // 各维度分数 (0~100)
    float CompletionScore = 0;      // 任务完成度
    float TeammateScore = 0;        // 队友管理
    float CombatScore = 0;          // 战斗效率
    float SurvivalScore = 0;        // 自保表现
    float ResourceScore = 0;        // 资源管理

    float TotalScore = 0;           // 加权总分
    EMissionRank Rank = EMissionRank::D;
};

USTRUCT(BlueprintType)
struct FMissionResultData
{
    GENERATED_BODY()

    // === 基础 ===
    float MissionTimeSec = 0;
    bool bPrimaryObjectiveCompleted = false;

    // === 战斗 ===
    int32 EnemiesKilled = 0;
    int32 TotalEnemies = 0;
    int32 ShotsFired = 0;
    int32 ShotsHit = 0;
    int32 AmmoUsed = 0;
    int32 TotalAmmo = 0;

    // === 队友 ===
    bool bTeammateA_Alive = false;
    bool bTeammateB_Alive = false;
    float TeammateA_HealthPercent = 0;
    float TeammateB_HealthPercent = 0;
    int32 OrdersGiven = 0;
    int32 FriendlyFireHits = 0;

    // === 玩家 ===
    int32 PlayerHitsTaken = 0;
    int32 RoomsCleared = 0;
    int32 TotalRooms = 0;

    // === 评价结果 ===
    FMissionScoreBreakdown Breakdown;
    FString RankComment;
};
```

## 四、评语库

### 等级评语

| 等级 | 评语 |
|------|------|
| **S** | 无懈可击的指挥！队友毫发无损，敌人全数歼灭，你是真正的精英指挥官。 |
| **A** | 出色的表现！指挥流畅，战斗高效，距离精英只有一步之遥。 |
| **B** | 合格的指挥，但还有提升空间。注意队友状态和弹药管理。 |
| **C** | 任务虽然完成，但伤亡过重。建议在训练屋中加强练习。 |
| **D** | 任务失败。好好总结教训，调整战术再试一次。 |

### 条件触发评语（追加显示）

```
无伤通关        → "全程无伤，教科书般的操作！"
弹药利用率>80%  → "每一发子弹都没有浪费，真正的老兵。"
队友全部阵亡    → "你是唯一幸存者...有时候队友也需要你的指挥。"
误伤队友        → "注意你的射击线！队友不是敌人。"
完美清房        → "每个房间都清理干净，没有任何遗漏。"
极速通关        → "闪电般的行动，敌人来不及反应。"
```

## 五、结算画面 VR UI 设计

### 核心理念：三级递进展示

VR 中文字多了看不清，满了不想看。解决方案是让玩家**自己决定看多深**。

### 第一层：进结算画面 — 只看等级（2秒读完）

```
┌──────────────────────────────┐
│                              │
│          ★ S ★               │  ← 巨大的等级，占画面 60%
│       精英指挥官              │
│                              │
│     用时 4分28秒             │  ← 只放 3 个关键数字
│     击杀 5/5  无伤           │
│                              │
│    [查看详情]                 │  ← 触发第二层
│    [回到主页]                 │
└──────────────────────────────┘
```

### 第二层：查看详情 — 5 条评分条（颜色一眼扫过）

当玩家伸手点击「查看详情」，展开：

```
  ┌────────────────────────────┐
  │ 详细评价                    │
  │                            │
  │ 任务完成度  ████████░░ 82% │  ← 颜色区分
  │ 队友管理    ██████░░░░ 64% │     绿色=好
  │ 战斗效率    █████████░ 94% │     黄色=一般
  │ 自身表现    ████████░░ 80% │     红色=差
  │ 资源管理    █████░░░░░ 52% │
  │                            │
  │ [展开数据] → 第三层         │
  │ [收起]                     │
  └────────────────────────────┘
```

### 第三层：查看具体数字（按需展开）

```
  ┌────────────────────────────┐
  │ 战斗数据                    │
  │  击杀敌人       5/5       │
  │  命中率         68%       │
  │  消耗弹药       23/100发  │
  │  开枪次数       34次      │
  │                            │
  │ 队友状态                   │
  │  队友A    ✅ 存活 92%血   │
  │  队友B    ✅ 存活 76%血   │
  │  下达指令   8次           │
  │                            │
  │ 玩家                       │
  │  受伤次数    1次           │
  │  清理房间    4/6          │
  └────────────────────────────┘
```

### 交互方式（手势操作）

| 操作 | 手势 |
|------|------|
| 查看详情 | 指向「查看详情」按钮 |
| 展开/收起数据 | 指向「展开数据」/「收起」 |
| 回到主页 | 握拳拉回（集合手势） |
| 再来一次 | 手掌前推（进入手势） |
```
