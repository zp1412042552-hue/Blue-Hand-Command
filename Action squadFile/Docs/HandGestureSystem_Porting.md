# ⚠️ 已废弃 — 手部姿势识别系统移植方案

> **废弃原因**：新方案（[TacticalGestureCommandSystem.md](TacticalGestureCommandSystem.md)）不再依赖 Oculus Hand Tools 插件的 `UHandPoseRecognizer` 组件。
>
> 旧方案需要从 Tiny Trouble VR 项目移植 Oculus 专用手势识别插件，并录制个人姿势模板。新方案采用 **特征点角度匹配 + 时间滤波**，基于通用 OpenXR 手部追踪，5 个手势均为天然大动作，无需插件移植、无需录制关卡。
>
> 本文档保留作为历史参考。

---

## 旧方案 vs 新方案

| 维度 | 旧方案（本文档） | 新方案 |
|------|-----------------|--------|
| 手势数量 | 7 个（选人×2 + 命令×5） | **5 个** |
| 依赖插件 | OculusHandTools / OculusUtils | **无**（通用 OpenXR） |
| 识别方式 | UHandPoseRecognizer 模板匹配 | 特征点角度匹配 + 时间滤波 |
| 姿势模板 | 需要录制关卡 + JSON 模板文件 | **无需录制** |
| 空间限制 | 手必须靠近面部 ≤ 30cm | **无限制** |
| 误触防护 | 空间触发区 | 状态机被动激活 + 0.5 秒时间过滤 |
| 破门操作 | 独立手势 ✊ | 合并到 👇（Raycast 命中 Door） |
| 禁火/警戒 | 两个独立手势 ✋ 和 🤚 | 合并为一个 ✋ |
| 集合操作 | 需先选人 | 👊 全局召回，无需选人 |
| 上下文感知 | 无 | ✅ 👇 根据 Raycast Tag 自动切换 3 种命令 |
| 多选/全选 | 不支持 | ✅ ①② 连做 |

## 旧内容摘要（仅供历史参考）

旧方案架构：
```
Oculus XR Plugin (手部追踪数据)
        ↓
UHandPoseRecognizer (左/右手各一个)
   └─ 加载姿势模板 (FHandPose)
   └─ 每帧计算手部角度误差
   └─ 返回置信度最高的姿势
        ↓
CommandGestureComponent (指挥手势组件)
   ├─ 空间模式管理（战斗区 vs 触发区）
   ├─ 战斗子模块：双拳瞄准 + 交叉射击检测
   ├─ 指挥子模块：7 个手势模板识别
   └─ 触发 HUD 反馈
        ↓
AI 队友 Blackboard（接收到命令 → Behavior Tree 执行）
```

旧方案 7 个手势：
| # | 手势 | 功能 |
|---|------|------|
| ① | ☝️ 伸 1 指 | 选中队友 A |
| ② | ✌️ 伸 2 指 | 选中队友 B |
| ③ | 👇 指向地面 | 移动到位置 |
| ④ | ✊ 握拳朝向门 | 破门 |
| ⑤ | ✋ 手掌竖起 | 加强警戒 |
| ⑥ | 🤚 手掌前推 | 禁止开火 |
| ⑦ | 👊 握拳拉回 | 集合保卫 |
