# AI 队友行为系统设计

## 一、设计哲学

- **两个 AI 队友首先是保护者，然后是执行者。**
- 不指挥时：在玩家 5 米范围内自由巡逻警戒。
- 指挥时：通过手势选择并下达命令，执行完毕后回到自由巡逻。
- 发现敌人：自动攻击，不需要玩家指挥。

---

## 二、3 层行为状态

```
┌──────────────────────────────────────────────┐
│ 第1层：自由巡逻（默认状态）                    │
│  队友在玩家 5 米范围内游走/警戒               │
│  面朝外部，保护玩家                           │
│  发现敌人 → 自动攻击                         │
├──────────────────────────────────────────────┤
│ 第2层：命令等待（玩家已选中队友）              │
│  玩家伸 1 根手指 持续 ≥ 1秒 → 选中队友A      │
│  玩家伸 2 根手指 持续 ≥ 1秒 → 选中队友B      │
│  选中的队友高亮 + 进入等待命令状态             │
├──────────────────────────────────────────────┤
│ 第3层：执行命令（玩家下达了具体指令）          │
│  执行完毕后 → 回到第1层                       │
└──────────────────────────────────────────────┘
```

---

## 三、命令列表（6 个手势 → 6 个命令）

| 手势 | 功能 | 队友行为 |
|------|------|---------|
| 1 根手指 ≥ 1秒 | 选择队友 A | 高亮队友A，进入命令等待 |
| 2 根手指 ≥ 1秒 | 选择队友 B | 高亮队友B，进入命令等待 |
| 握拳 + 朝向门 | 踢门突入 | 队友走到门前 → 踢门 → 进入房间搜索 → 反馈"安全"或"发现目标" |
| 指地面 | 移动到指定位置 | 队友走到手指指向的地面位置 → 原地警戒 |
| 手掌竖起 | 原地警戒 | 队友停在当前位置，面朝指定方向警戒 |
| 手掌前推 | 进入房间搜索 | 队友走进房间 → 遍历搜索点 → 反馈结果 |
| 握拳拉回 | 集合 | 队友走回玩家身边 → 恢复自由巡逻 |

## 四、Behavior Tree 逻辑（选择器优先级）

```
Root (Selector)
│
├── [1] 执行命令（最高优先级）
│   条件：CommandType != None
│   动作：执行对应命令
│   完成：CommandType = None，回到巡逻
│
├── [2] 战斗（次优先级）
│   条件：EnemyActor != None
│   动作：面向敌人 → 开火
│
├── [3] 自由巡逻（默认）
│   ├── 距离玩家 > 5米 → 走回玩家身边
│   └── 距离玩家 ≤ 5米 → 随机游走 / 站立警戒
```

---

## 五、数据通信（Blackboard 条目）

```
Blackboard 条目（每个队友独立）：
├── SelfActor               本身引用
├── PlayerActor             玩家引用
├── CommandType             (None / MoveTo / Watch / Breach / Search / Return)
├── TargetLocation          目标位置（指地面/踢门目标点）
├── TargetDoor              目标门（踢门时）
├── EnemyActor              当前敌人
└── bSelected               是否被玩家选中
```

手势组件只负责：**识别姿势 → 修改 Blackboard → Behavior Tree 自动响应**

## 六、关键技术决策

| 决策 | 方案 | 理由 |
|------|------|------|
| 寻路 | NavMesh（导航网格） | UE 内置，自动绕障，适用室内环境 |
| AI 决策 | Behavior Tree | UE 可视化节点搭建，无需手写复杂逻辑 |
| 数据通信 | Blackboard | UE 内置，与 Behavior Tree 天然配合 |
| 特殊动作 | Montage 动画（踢门） | 播放动画序列，不需要 IK 或物理 |
| 代码量 | 只需要写 CommandGestureComponent | AI 部分全用 UE 编辑器搭节点 |

## 七、房间搜索路径点（SearchPoints）

```
每个房间预先放置搜索点：
┌──────────────────────┐
│                      │
│  ①                   │  ① → 左侧角落
│           ②          │  ② → 中央
│                      │  ③ → 右侧角落
│                ③     │  ④ → 目标物附近
│                      │
│        ④            │
│                      │
└──────────┬───────────┘
         门
         
队友进入房间后按顺序遍历搜索点。
到达每个点 → 短暂停留 → 扫描周围 → 移动到下一个。
全部遍历完毕 → 反馈"区域安全"或"发现目标"。
```

## 八、队友保护玩家（自由巡逻策略）

```
① 采样玩家位置
② 计算离玩家距离
③ 如果 > 5 米 → 走回玩家身边
④ 如果 ≤ 5 米 → 选择一个新的巡逻点（在 5 米半径内）
⑤ 走到巡逻点 → 站定 → 朝向外部警戒 → 停留数秒
⑥ 重复
```

警戒时面朝外部（面朝远离玩家的方向），确保玩家背后和侧面有人看着。
