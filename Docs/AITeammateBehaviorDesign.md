# AI 队友行为系统设计

## 一、设计哲学

- **两个 AI 队友首先是保护者，然后是执行者。**
- 不指挥时：在玩家 5 米范围内自由巡逻警戒。
- 指挥时：通过手势选择并下达命令，执行完毕后回到自由巡逻。
- 发现敌人：自动攻击，不需要玩家指挥。

---

## 二、3 层行为状态

```
┌──────────────────────────────────────────────┐
│ 第1层：自由巡逻（默认状态）                    │
│  队友在玩家 5 米范围内游走/警戒               │
│  面朝外部，保护玩家                           │
│  发现敌人 → 自动攻击                         │
├──────────────────────────────────────────────┤
│ 第2层：命令等待（玩家用手势选中了队友）          │
│  玩家右手靠近面部 ≤ 30cm → 进入手势触发区     │
│  伸 1 指 ≥ 0.5秒 → 队友A 高亮选中            │
│  伸 2 指 ≥ 0.5秒 → 队友B 高亮选中            │
│  选中后队友停止移动，面向玩家等待命令          │
│  手放下/离开触发区 → 取消选中                 │
├──────────────────────────────────────────────┤
│ 第3层：执行命令（玩家下达了具体指令）          │
│  执行完毕后 → 回到第1层                       │
└──────────────────────────────────────────────┘
```

## 三、选人方式与命令列表（5 个手势）

### 3.1 选人方式：手指手势（非视线）

玩家通过**伸几根手指**来选择队友，不需要转头看谁：

```
右手靠近面部 ≤ 30cm（触发区）→ 手势识别开启
  伸 1 根食指 ≥ 0.5秒 → 队友 A 高亮（蓝色轮廓光）→ 选中
  伸 2 根手指 ≥ 0.5秒 → 队友 B 高亮（黄色轮廓光）→ 选中
  手放下/离开触发区 → 取消选中
```

> 触发区 = 以 HMD 为球心、半径 30cm 的球体。手进入这个区域后手势识别才生效（详见 [HandGestureSystem_Porting.md](HandGestureSystem_Porting.md) 第 4.2 节）。

### 3.2 7 个手势总表

| # | 手势 | 所属类别 | 功能 | 队友行为 |
|---|------|---------|------|---------|
| ① | ☝️ 伸 1 根食指 | 选人 | **选中队友 A** | 队友 A 蓝色高亮，面向玩家等待命令 |
| ② | ✌️ 伸 2 根手指 | 选人 | **选中队友 B** | 队友 B 黄色高亮，面向玩家等待命令 |
| ③ | 👇 食指指向地面 | 命令 | **移动到位置** | 队友走到手指指向的地面位置 → 原地警戒 |
| ④ | ✊ 握拳朝向门 | 命令 | **破门** | 队友走到门前 → 踢门 → 进入房间搜索 → 反馈安全或发现目标 |
| ⑤ | ✋ 手掌竖起 | 命令 | **加强警戒** | 队友停在当前位置，面朝指定方向持续警戒 |
| ⑥ | 🤚 手掌前推 | 命令 | **禁止开火** | 队友停止射击，只观察不攻击，直到收到新命令解除 |
| ⑦ | 👊 握拳拉回胸口 | 命令 | **集合保卫** | 队友走回玩家左右两侧（1~2米），面朝外部警戒，恢复自由巡逻 |

> 命令手势 ③~⑦ 只在**已选中队友**的状态下生效。未选中时做这些手势无反应。

## 四、Behavior Tree 逻辑（选择器优先级）

```
Root (Selector)
│
├── [1] 执行命令（最高优先级）
│   条件：CommandType != None
│   动作：执行对应命令
│   完成：CommandType = None，回到巡逻
│
├── [2] 战斗（次优先级）
│   条件：EnemyActor != None
│   动作：面向敌人 → 开火
│
├── [3] 自由巡逻（默认）
│   ├── 距离玩家 > 5米 → 走回玩家身边
│   └── 距离玩家 ≤ 5米 → 随机游走 / 站立警戒
```

## 五、数据通信（Blackboard 条目）

```
Blackboard 条目（每个队友独立）：
├── SelfActor               本身引用
├── PlayerActor             玩家引用
├── CommandType             (None / MoveTo / Breach / Watch / HoldFire / Return)
├── TargetLocation          目标位置（指地面/踢门目标点）
├── TargetDoor              目标门（踢门时）
├── EnemyActor              当前敌人
├── bSelected               是否被玩家视线选中
└── bHoldFire               是否禁止开火
```

手势组件只负责：**识别姿势 → 修改 Blackboard → Behavior Tree 自动响应**

视线选择组件只负责：**判断玩家视线方向 → 修改 bSelected / EnemyActor**

## 六、关键技术决策

| 决策 | 方案 | 理由 |
|------|------|------|
| 寻路 | NavMesh（导航网格） | UE 内置，自动绕障，适用室内环境 |
| AI 决策 | Behavior Tree | UE 可视化节点搭建，无需手写复杂逻辑 |
| 数据通信 | Blackboard | UE 内置，与 Behavior Tree 天然配合 |
| 特殊动作 | Montage 动画（踢门） | 播放动画序列，不需要 IK 或物理 |
| 代码量 | 只需要写 CommandGestureComponent | AI 部分全用 UE 编辑器搭节点 |

## 七、房间搜索路径点（SearchPoints）

```
每个房间预先放置搜索点：
┌──────────────────────┐
│                      │
│  ①                   │  ① → 左侧角落
│           ②          │  ② → 中央
│                      │  ③ → 右侧角落
│                ③     │  ④ → 目标物附近
│                      │
│        ④            │
│                      │
└──────────┬───────────┘
         门
         
队友进入房间后按顺序遍历搜索点。
到达每个点 → 短暂停留 → 扫描周围 → 移动到下一个。
全部遍历完毕 → 反馈"区域安全"或"发现目标"。
```

## 八、队友保护玩家（自由巡逻策略）

```
① 采样玩家位置
② 计算离玩家距离
③ 如果 > 5 米 → 走回玩家身边
④ 如果 ≤ 5 米 → 选择一个新的巡逻点（在 5 米半径内）
⑤ 走到巡逻点 → 站定 → 朝向外部警戒 → 停留数秒
⑥ 重复
```

警戒时面朝外部（面朝远离玩家的方向），确保玩家背后和侧面有人看着。
