# 手部姿势录制关卡 — HandPoseRecordingLevel

## 一、设计理念

录制手势模板是整个手势识别系统的**第一步**，也是最关键的一步。模板质量直接决定识别准确率。

Tiny Trouble 的做法是在 VR Preview 中直接录制（有游戏场景干扰），而我们专门建一个**纯黑录制关卡**：

```
┌──────────────────────────────────────┐
│                                      │
│          纯黑背景 (无任何物体)          │
│                                      │
│                                      │
│        ✋  自发光蓝色手掌              │
│        (玩家能看到自己的手)            │
│                                      │
│                                      │
│      ┌──────────────────────┐        │
│      │  姿势录制提示屏幕      │        │
│      │  当前：第 1/5 个手势   │        │
│      │  请摆出「选择队友A」   │        │
│      │  ☝️ 伸出食指，其余握拳  │        │
│      │                      │        │
│      │  倒计时：5 秒         │        │
│      └──────────────────────┘        │
│                                      │
└──────────────────────────────────────┘
```

**优点**：
- 无视觉干扰，玩家专注摆姿势
- 自发光手在纯黑背景下对比度最高，追踪最稳定
- 录制数据干净，排除场景光照对手部追踪的影响
- 可快速重复录制，直到满意

## 二、关卡布局

### 2.1 环境设置

| 项目 | 设置 |
|------|------|
| 背景 | 纯黑 (0,0,0)，无任何网格/物体 |
| 光照 | 无任何光源（全靠手部自发光材质） |
| 地面 | 无，让玩家站在虚空 |
| 大小 | 极小，只有玩家初始位置 + HUD 屏幕 |

### 2.2 玩家初始位置

玩家在关卡中只有一个初始点。不需要移动，不需要武器，不需要队友。

### 2.3 唯一交互元素

一个 **HUD 屏幕**（World Space 的 Widget 或者 3D TextActor），位置在玩家正前方约 2 米处，稍微偏上（约 1.7 米高），让玩家抬手时能看到自己的手在屏幕和视线之间。

```
        ┌──────────────────────┐
        │                      │
        │   ← 屏幕在前上方      │
        │                      │
        │      玩家             │
        │      (手在眼前)       │
        │                      │
        └──────────────────────┘
```

## 三、录制流程

### 3.1 完整流程（5 个手势依次录制）

```
关卡加载
    │
首次进入 → 显示欢迎信息 + 手势列表
    │
玩家用指尖指向「开始录制」按钮（或用键盘按 1）
    │
┌──────────────────────────────────┐
│ 第 1 轮：录制「选择队友A」         │
│                                  │
│ ① 屏幕显示：                     │
│   「第 1/5 个手势：移动到位置」     │
│   「请摆出姿势：伸出食指，其余握拳」 │
│   「准备好后，手势将自动开始倒计时」 │
│   或者：按空格键开始倒计时         │
│                                  │
│ ② 玩家摆出姿势，按空格/稳定 1秒   │
│                                  │
│ ③ 倒计时 5、4、3、2、1           │
│   期间手部姿势保持不动            │
│                                  │
│ ④ 录制中... (采集数据)           │
│   进度条显示                      │
│                                  │
│ ⑤ 录制完成                       │
│   「✅ 选择队友A 录制完成」         │
│   「共采集 120 帧样本」            │
│   「按 2 继续下一个手势」          │
│   或自动跳到下一个                │
└──────────────────────────────────┘
    │
┌──────────────────────────────────┐
│ 第 2 轮：录制「选择队友B」         │
│  ...（同上）                      │
└──────────────────────────────────┘
    │
...（共 6 轮）
    │
┌──────────────────────────────────┐
│ 全部录制完成                      │
│                                  │
│ 「🎉 所有 5 个手势录制完成！」      │
│ 「模板已保存到：                   │
│  Saved/HandPoseGestures/         │
│  BlueHandCommandGestureProfile.json」│
│                                  │
│ [返回主页]  [重新录制某单个手势]    │
└──────────────────────────────────┘
```

### 3.2 触发方式

录制关卡从 **Home 关卡**进入。在 Home 中有一个选项：

- **首次使用** → 自动提示进入录制关卡
- **已有模板** → 跳过录制，直接进入游戏
- **重新录制** → 设置中提供「重录手势」按钮

### 3.3 两种启动录制的方式

| 方式 | 适用场景 | 操作 |
|------|---------|------|
| 键盘按键（PC 预览） | 开发阶段调试 | 按 `1`~`6` 选择槽位，按 `Space` 开始录制 |
| 手势/指尖点击（Quest） | 头盔内录制 | 用指尖指向屏幕上的「开始」按钮 |

- 震动反馈确认

## 五、加载与启动逻辑

### 5.1 关卡信息

| 项目 | 内容 |
|------|------|
| 关卡名称 | `HandPoseRecordingLevel` |
| 包路径 | `/Game/Maps/HandPoseRecording` |
| 加载方式 | 从 Home 通过 `FBlueHandCommandLevelRequest` 跳转 |
| 存储路径 | `Saved/HandPoseGestures/BlueHandCommandGestureProfile.json` |

### 5.2 跳转逻辑（修改 Architecture_TreeDesign 的流程）

```
Home (Settings → 手势录制)
    │
    └→ RequestLevelTransition(TargetLevelName="HandPoseRecording")
         TargetPhase = None (录制关卡不属于任何游戏阶段)
         MetaData = (无)
              │
              ↓
    HandPoseRecordingLevel (录制完成)
              │
              ↓
    RequestLevelTransition(TargetLevelName="Home")
```

### 5.3 录制模式选择

玩家进入录制关卡的三种方式：

| 入口 | 触发条件 | 行为 |
|------|---------|------|
| 首次启动 | 用户尚未录制过任何手势（SaveGame 检测） | 自动弹出提示，询问是否进入录制关卡 |
| 设置菜单 | 用户主动选择「重新录制手势」 | 从 Home 跳转到录制关卡 |
| 手势识别失败 | 运行时所有手势置信度持续低于阈值 | 提示「建议重新录制手势模板」 |

## 六、自发光手部材质

### 6.1 材质要求

在纯黑关卡中玩家需要能看到自己的手：

| 要求 | 说明 |
|------|------|
| 材质类型 | Unlit（不受光照影响） |
| 自发光颜色 | 蓝色（项目主题色，RGB 0.0~0.6 的蓝白色） |
| 半透明 | 轻微半透明（0.8 不透明度），能看到手指重叠 |
| 轮廓光 | 手指边缘稍微亮一点（Fresnel 效果） |
| 无阴影 | 纯黑场景不需要阴影计算 |

### 6.2 材质节点（参考）

```
自发光颜色 = 蓝白色 (0.3, 0.5, 1.0)
混合模式 = Translucent
 shading Model = Unlit

Fresnel: 边缘更亮 (Outline)
  → 手指轮廓清晰可见

与默认手部材质的区别：
  默认材质 → 依赖场景光照
  录制材质 → 纯自发光，不依赖任何光源
```

## 七、对 5 个手势的录制建议

Tiny Trouble 每个姿势录了 **108~110 帧**（1.25 秒 × 90fps 大约 112 帧）。

而我们录 5 秒，会采集更多样本（约 300 帧），平均值更稳定。

| # | 手势 | 姿势描述 | 录制要点 |
|---|------|---------|---------|
| ① | 👇 移动到位置 | 右手食指指向地面方向 | 手臂自然伸直，食指朝下，其余四指握拳 |
| ② | ✊ 破门 | 右手握拳，拳眼朝前 | 握紧拳头，手臂略弯，拳眼对准门的方向 |
| ③ | ✋ 加强警戒 | 右手手掌打开竖起（Stop 手势） | 五指自然分开，掌心朝前，手腕立起 |
| ④ | 🤚 禁止开火 | 单手或双手手掌前推 | 掌心朝前，手臂平伸，像挡住什么东西 |
| ⑤ | ✊ 集合保卫 | 右手握拳快速拉回胸前 | 握紧拳头，手臂快速弯回胸口，类似拉绳子 |

## 九、与 Tiny Trouble 录制方式的区别

| 项目 | Tiny Trouble | Blue-Hand-Command |
|------|-------------|-------------------|
| 关卡 | 在游戏场景中录制 | 专用纯黑录制关卡 |
| 背景 | 有场景物体 | 纯黑，零干扰 |
| 倒计时 | 3 秒 | 5 秒（更多准备时间） |
| 采集时长 | 1.25 秒 | 5 秒（更多样本，更稳定） |
| 每手势样本 | ~110 帧 | ~300 帧（90fps × 5s × 2/3 稳定段） |
| 录制触发 | 键盘 R 键 | 键盘 Space 或指尖点击屏幕按钮 |
| 界面 | HUD 仅显示短状态文字 | 全屏 Widget 显示详细说明 |
| 录制顺序 | 手动选槽位，自由顺序 | 引导式 1→6 依次录制 |
| 手部显示 | 默认手部材质（依赖场景光） | 自发光材质（纯黑可见） |
| 重录 | 同槽位覆盖 | 支持单个手势重录 + 全部重录 |
